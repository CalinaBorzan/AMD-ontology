# DL-Learner Experiments

This folder contains DL-Learner configuration files for learning OWL class expressions from the enriched AMD ontology.

## Requirements

- DL-Learner 1.5.0 (run via Docker)
- `AMD.owl` must be present in the parent directory (generated by `enrich_owl.py`)

## Running

```bash
# Start Docker container with DL-Learner
docker run -it -v $(pwd)/..:/workspace dllearner

# Inside container
java -jar dllearner-1.5.0/bin/cli dl_learner/<config_file>.conf
```

## Experiments

### 1. `dl_learner_wet_amd.conf`
**Question**: What characterises WetAMD abstracts vs DryAMD/generic abstracts?
- Positives: 55 abstracts typed as WetAMD
- Negatives: 37 abstracts typed as DryAMD or Disease
- Result: `WetAMD (100%)` — trivial, type label is perfect discriminator

### 2. `dl_learner_antivegf.conf`
**Question**: What characterises abstracts involving anti-VEGF treatments?
- Positives: 38 abstracts with `involvesTreatment → AntiVEGFTherapy`
- Negatives: 29 abstracts with other treatments
- Result: `involvesTreatment some AntiVEGFTherapy (100%)` — validates enrichment correctness

### 3. `dl_learner_treatment.conf`
**Question**: What distinguishes AntiVEGF from other treatment categories?
- Positives: 12 AntiVEGFTherapy instances
- Negatives: 23 instances from other treatment classes

### 4. `dl_learner_mechanistic.conf`
**Question**: What characterises mechanistic studies (treatment + biomarker/diagnostic) vs pure intervention trials (treatment only)?
- Positives: 42 abstracts with treatment AND biomarker/diagnostic
- Negatives: 25 abstracts with treatment only
- Result: `involvesTreatment some (treats max 4 Thing) (65.67%)` — non-trivial, shows structural pattern discovery

## Key Finding

CELOE produces trivial solutions when the positive/negative split is directly encoded as an OWL property. Non-trivial learning requires classification criteria external to the ontology (e.g., expert annotations). See paper for full discussion.
